# 论文提纲与图表清单（严格贴合当前项目）

## 一、论文提纲（建议结构）

### 1. 引言
- 研究背景：法律大模型应用中的幻觉风险与可解释性要求。
- 问题陈述：仅靠通用LLM难以满足司法辅助的证据可追溯性。
- 研究目标：构建本地知识优先、支持法官修订闭环的法律助手系统。
- 本文贡献：
  1. 本地案例+法条联合检索分析框架；
  2. 法官修订标准并列优先注入机制；
  3. 动态检索+静态映射融合（Hybrid）修订依据生成；
  4. 严格评测脚本与可复现实验流程。

### 2. 相关工作
- 法律问答与司法辅助系统
- 检索增强生成（RAG）在法律场景应用
- 人在回路（Human-in-the-loop）与可解释AI

### 3. 系统设计
- 3.1 总体架构：UI层、检索层、法条层、修订层、评测层
- 3.2 数据层：
  - 案例库：2001条
  - 法条库：452条
  - 静态映射：618罪名
- 3.3 检索层：
  - BGE编码、FAISS内积索引、阈值过滤与重试
- 3.4 修订层：
  - 法官操作台输入结构
  - Hybrid依据生成（动态优先 + 静态兜底）
- 3.5 约束层：
  - 本地知识优先提示词
  - 法官标准与本地证据冲突处理原则

### 4. 方法细节
- 4.1 向量检索与相似案例召回
- 4.2 法条编号归一与原文回查
- 4.3 法官修订标准注入与多轮对话记忆
- 4.4 动态/静态/混合映射策略定义

### 5. 实验设计
- 5.1 数据与预处理
- 5.2 评测任务
  - 检索任务：Hit@K, Precision@K, MRR@K, nDCG@K
  - 映射任务：fact/law/example coverage与信息量
- 5.3 对比设置
  - Dynamic only
  - Static only
  - Hybrid
  - （可选）w/o judge standard

### 6. 实验结果与分析
- 6.1 阈值扫描结果（scan）
- 6.2 严格检索结果（strict）
- 6.3 映射模式对比结果
- 6.4 案例研究（修订前后对比）
- 6.5 失败案例与误差来源

### 7. 讨论
- 工程可行性与部署价值
- 风险与局限（同域评测偏乐观、标签噪声、法条匹配边界）
- 改进方向（专家标注、跨域测试、证据图谱）

### 8. 结论
- 总结系统在司法辅助场景中的价值与边界。
